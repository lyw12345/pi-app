# 管理对话与分支（pi-web v1.1）

本文说明 pi-web 里与 pi CLI 共用的会话文件能力，不涉及 RPC 术语。

## 会话标题

- 在左侧会话列表中重命名；名称写入 `~/.pi/agent/sessions/` 下的 `.jsonl`（`session_info` 条目）。
- 聊天顶栏显示同一名称；刷新后仍保留。
- 请只用侧栏重命名，避免其它入口重复写入。

## 从这里另开一版（Fork）

- 悬停在某条**用户消息**上，点 **从这里另开一版**。
- 会新建**独立**的 `.jsonl` 文件，并在侧栏显示为子会话。
- 与「在当前文件里分支」不同。

## 复制为新对话（Clone）

- 在输入栏工具区点 **复制为新对话**。
- 把**当前分支路径**整段复制到新的会话文件，并自动打开新会话。
- 适合「保留现状、另开一条线继续」。

## 会话内分支

- 顶栏 **分支** 面板可切换同文件内的不同路径（Continue / 编辑分支）。
- 可选 **切换前先总结**（默认关）：离开当前分支前，用模型生成摘要并写入时间线。

## 已整理（压缩）

- 自动或手动整理后，时间线会出现 **已整理** 或 **分支摘要** 块，可展开查看全文。
- 不显示 token 统计等技术字段。

## 与 CLI 一致

同一工作目录（`cwd`）下，pi-web 与 `pi` 命令行读写同一批会话文件；在任一端新建、重命名、Fork 后，刷新另一侧即可看到。
